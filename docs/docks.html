1. Wprowadzenie

BitBudCoin (BBC) to lekki, szybki i w pełni otwarty blockchain Proof‑of‑Work zbudowany w JavaScript. Sieć jest zdecentralizowana, każdy może uruchomić własny node, kopać bloki i wysyłać transakcje.

2. Architektura

BBC składa się z kilku głównych modułów:

  • Blockchain — struktura bloków
  • Block — pojedynczy blok
  • Transaction — podpisane transakcje ECDSA
  • Mempool — oczekujące transakcje
  • Wallet — generowanie kluczy i podpisy
  • P2P — synchronizacja między węzłami
  • API — REST dla użytkowników

3. API Node'a

GET /info

Zwraca podstawowe informacje o node.

GET /chain

Zwraca cały blockchain.

GET /block/:id

Zwraca pojedynczy blok.

GET /balance/:address

Zwraca saldo adresu.

GET /mempool

Zwraca oczekujące transakcje.

POST /transaction

Dodaje podpisaną transakcję do mempoola.

{
  "from": "BbC123...",
  "to": "BbC456...",
  "amount": 10,
  "fee": 0.001,
  "fromPublicKey": "...",
  "signature": "..."
}

POST /mine

Kopie nowy blok.

4. Portfel BBC

Portfel generuje klucze ECDSA secp256k1:

privateKey → hex
publicKey  → hex
address    → BbC + sha256(publicKey).slice(0,40)

Podpisy transakcji:

hash = sha256(JSON.stringify(payload))
signature = key.sign(hash).toDER("hex")

5. Mining

BBC używa klasycznego Proof‑of‑Work:

while(hash.substring(0, difficulty) !== "0".repeat(difficulty)) {
    nonce++;
    hash = sha256(blockData + nonce);
}

6. P2P

Węzły komunikują się przez WebSocket:

  • CHAIN — synchronizacja chainu
  • TRANSACTION — broadcast transakcji
  • NEW_BLOCK — broadcast nowego bloku
